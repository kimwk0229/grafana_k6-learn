k6-Learn은 Grafana k6 성능 테스트 도구를 학습하고 가르치기 위한 오픈소스 교육 자료 저장소입니다. 슬라이드 프레젠테이션, 모듈식 학습 콘텐츠, 워크샵 자료를 모두 포함하고 있습니다.

1슬라이드 제작

reveal.js 기반의 인터랙티브 슬라이드를 직접 커스터마이징해서 프레젠테이션을 만들 수 있습니다.

2워크샵 진행

30분 발표부터 3일 워크샵까지 다양한 형태로 활용할 수 있는 모듈 구성입니다.

3k6 학습

성능 테스트 원칙부터 k6 고급 기능까지 단계별로 학습할 수 있는 체계적인 커리큘럼입니다.

4발표 자료

k6에 대한 발표나 강연을 준비할 때 기반 자료로 활용할 수 있습니다.

ℹ️
외부 리소스 추가적인 학습 자료로 k6 스타터 슬라이드 덱과 k6 OSS 워크샵도 활용할 수 있습니다.

📚모듈 I — 성능 테스트 원칙

성능 테스트의 기본 개념과 원칙을 다루는 모듈입니다. k6를 사용하기 전에 알아야 할 핵심 이론을 포함합니다.

01
성능 테스트란 무엇인가? 성능 테스트의 정의, 필요성, 흔한 오해 해소. 사용자 경험 개선, 예기치 못한 수요 대비, 인프라 최적화 등의 목적을 다룹니다. 퀴즈 포함. Modules/I-Performance-testing-principles/01-Introduction-to-Performance-Testing.md
02
프론트엔드 vs 백엔드 성능 테스트 프론트엔드(Core Web Vitals, LCP, FID, CLS)와 백엔드(API, 서버) 성능 테스트의 차이점과 각각의 접근 방식을 비교합니다. Modules/I-Performance-testing-principles/02-Frontend-vs-backend-performance-testing.md
03
부하 테스트 (Load Testing) 다양한 부하 테스트 유형(스모크, 부하, 스트레스, 스파이크, 소크, 브레이크포인트) 과 각 시나리오의 목적과 특성을 설명합니다. Modules/I-Performance-testing-principles/03-Load-Testing.md
04
부하 테스트 프로세스 개요 테스트 목표 설정, 스크립트 작성, 실행 환경 구성, 결과 분석까지의 전체 부하 테스트 수행 프로세스를 단계별로 안내합니다. Modules/I-Performance-testing-principles/04-High-level-overview-of-the-load-testing-process.md

🔧모듈 II — k6 기초 (Foundations)

k6 OSS를 처음 시작하는 사용자를 위한 기초 모듈입니다. 설치부터 결과 해석, 스크립트 작성까지 단계별로 학습합니다.

01
k6 OSS 시작하기 k6 설치, 첫 번째 스크립트 작성(k6/http import, default function), Hello World 실행. k6 run test.js 명령어로 기본 HTTP POST 테스트 수행. Modules/II-k6-Foundations/01-Getting-started-with-k6-OSS.md
02
k6 CLI k6 커맨드라인 인터페이스의 주요 명령어, 플래그(--vus, --duration, --iterations), 환경 변수 설정 방법을 다룹니다. Modules/II-k6-Foundations/02-The-k6-CLI.md
03
k6 결과 이해하기 테스트 종료 후 요약 보고서 해석: http_req_duration, http_req_failed, iterations 등 내장 메트릭의 의미와 분석 방법. Modules/II-k6-Foundations/03-Understanding-k6-results.md
04
체크(Checks) 추가하기 check() 함수를 사용해 응답 상태 코드, 응답 바디, 헤더 등을 검증하는 방법. 체크 실패는 테스트를 중단하지 않고 통계로 수집됩니다. Modules/II-k6-Foundations/04-Adding-checks-to-your-script.md
05
Think Time — sleep() 추가 실제 사용자 행동을 시뮬레이션하기 위한 sleep() 함수 사용법. 고정 대기, 랜덤 대기(Math.random())로 현실적인 부하 패턴을 만드는 방법. Modules/II-k6-Foundations/05-Adding-think-time-using-sleep.md
06
k6 부하 테스트 옵션 옵션 설정 우선순위(스크립트 > 환경변수 > CLI 플래그), vus, duration, iterations, stages(램프업/다운) 설정 방법. Modules/II-k6-Foundations/06-k6-Load-Test-Options.md
07
임계값(Thresholds)으로 테스트 기준 설정 오류율(http_req_failed), 응답 시간(http_req_duration p(95)), 체크 성공률 기반 임계값 정의. abortOnFail로 임계값 위반 시 테스트 자동 중단. Modules/II-k6-Foundations/07-Setting-test-criteria-with-thresholds.md
08
k6 결과 출력 옵션 CSV, JSON, InfluxDB, Prometheus, Grafana Cloud 등 다양한 출력 형식. CI/CD 파이프라인 통합을 위한 결과 활용 방법. Modules/II-k6-Foundations/08-k6-results-output-options.md
09
k6 스크립트 녹화 브라우저 확장 프로그램(k6 Browser Recorder)이나 HAR 파일을 사용해 실제 사용자 행동을 녹화하고 k6 스크립트로 변환하는 방법. Modules/II-k6-Foundations/09-Recording-a-k6-script.md

기초 예제: 첫 번째 k6 스크립트

import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '30m', target: 100 },  // 30분간 100 VU로 램프업
    { duration: '1h',  target: 100 },  // 1시간 유지
    { duration: '5m',  target: 0   },  // 5분간 램프다운
  ],
  thresholds: {
    http_req_failed:   ['rate<=0.05'],     // 오류율 5% 이하
    http_req_duration: ['p(95)<=5000'],   // 95% 응답시간 5초 이하
    checks:            ['rate>=0.99'],    // 체크 성공률 99% 이상
  },
};

export default function() {
  const res = http.post('https://httpbin.test.k6.io/post', 'Hello world!');
  check(res, {
    'status is 200': (r) => r.status === 200,
    'body includes hello': (r) => r.body.includes('Hello world!'),
  });
  sleep(Math.random() * 5);
}

⚙️모듈 III — k6 중급 (Intermediate)

k6 기초를 마스터한 후 더 복잡한 시나리오와 고급 기능을 다루는 중급 모듈입니다.

01
k6 스크립트 디버깅 --http-debug 플래그, console.log(), 응답 바디 검사 등 스크립트 문제를 진단하는 다양한 디버깅 기법. Modules/III-k6-Intermediate/01-How-to-debug-k6-load-testing-scripts.md
02
동적 상관관계(Dynamic Correlation) 세션 토큰, CSRF 토큰 등 동적 값을 응답에서 추출하여 다음 요청에 재사용하는 상관관계 처리 방법. JSON/정규식 파싱 활용. Modules/III-k6-Intermediate/02-Dynamic-correlation-in-k6.md
03
워크로드 모델링 실제 사용자 행동 패턴을 테스트에 반영하는 방법. 동시 사용자(VU), 도착률(arrival rate), 대기 시간 분포를 활용한 현실적 부하 모델 설계. Modules/III-k6-Intermediate/03-Workload-modeling.md
04
테스트 데이터 추가 CSV 파일에서 SharedArray로 데이터 로드, 무작위/순환 데이터 선택, 환경 변수를 통한 데이터 주입으로 더 현실적인 테스트 데이터 관리. Modules/III-k6-Intermediate/04-Adding-test-data.md
05
병렬 요청 (Parallel Requests) http.batch()로 여러 요청을 동시에 보내는 방법. 브라우저가 리소스를 병렬로 로드하는 패턴을 시뮬레이션합니다. Modules/III-k6-Intermediate/05-Parallel-requests-in-k6.md
06
그룹(Groups)과 태그(Tags)로 코드 구성 group()으로 관련 요청을 논리적 트랜잭션으로 묶고, tags로 메트릭을 세분화하여 분석하기 쉬운 구조로 스크립트를 구성합니다. Modules/III-k6-Intermediate/06-Organizing-code-in-k6-by-transaction_groups-and-tags.md
07
Setup과 Teardown 함수 테스트 전(setup())에 데이터 준비, 인증 토큰 획득을 하고, 테스트 후(teardown())에 정리 작업을 수행하는 라이프사이클 함수 활용. Modules/III-k6-Intermediate/07-Setup-and-Teardown-functions.md
08
Executor로 부하 프로파일 설정 7가지 executor 유형으로 다양한 부하 패턴 구현: shared-iterations, per-vu-iterations, constant-vus, ramping-vus, constant-arrival-rate, ramping-arrival-rate, externally-controlled. Modules/III-k6-Intermediate/08-Setting-load-profiles-with-executors.md
09
시나리오(Scenarios)로 워크로드 모델링 여러 사용자 흐름을 독립적인 시나리오로 분리하여 동시에 실행. startTime으로 시나리오 지연 시작, exec로 다른 함수 실행 지정. Modules/III-k6-Intermediate/09-Workload-modeling-with-scenarios.md
10
실행 컨텍스트 변수 활용 __VU(가상 사용자 번호), __ITER(반복 횟수), __ENV(환경 변수) 등 내장 컨텍스트 변수를 활용해 VU별로 다른 동작을 구현합니다. Modules/III-k6-Intermediate/10-Using-execution-context-variables.md
11
커스텀 메트릭 생성 및 활용 Counter, Gauge, Rate, Trend 4가지 커스텀 메트릭 유형으로 비즈니스 로직에 맞는 측정값(예: 주문 성공률, 장바구니 금액)을 추적합니다. Modules/III-k6-Intermediate/11-Creating-and-using-custom-metrics.md

Executor 비교표

Executor제어 방식주요 용도
shared-iterations총 반복 횟수고정 작업량을 여러 VU가 나눠서 처리
per-vu-iterationsVU당 반복 횟수각 VU가 동일한 횟수만큼 실행
constant-vus동시 VU 수일정 시간 동안 고정 VU 유지
ramping-vusVU 수 변화램프업/다운 패턴, 스트레스 테스트
constant-arrival-rate초당 요청 수(RPS)고정 처리율 유지, 용량 테스트
ramping-arrival-rate변화하는 RPS점진적 부하 증가, 파괴 테스트

시나리오 예제

export const options = {
  scenarios: {
    // 시나리오 1: 상품 검색 (백그라운드 활동)
    users_browsing_products: {
      executor:    'shared-iterations',
      env:         { WORKFLOW: 'browsing' },
      vus:         3,
      iterations:  10,
      maxDuration: '10s',
    },
    // 시나리오 2: 구매 플로우 (2초 후 시작)
    users_buying_products: {
      executor:  'per-vu-iterations',
      exec:      'usersBuyingProducts',
      env:       { WORKFLOW: 'buying' },
      startTime: '2s',
      vus:       3,
      iterations:1,
    },
  },
};

export default function() {
  console.log(`[VU: ${__VU}, iter: ${__ITER}] 상품 검색 중...`);
}

export function usersBuyingProducts() {
  console.log(`[VU: ${__VU}, iter: ${__ITER}] 구매 완료!`);
}

🛠️기술 스택

reveal.js 4.5

HTML/Markdown 기반의 웹 슬라이드 프레임워크. 키보드 네비게이션, 발표자 노트, PDF 내보내기 지원.

Vite 5.0

차세대 프론트엔드 빌드 도구. HMR(Hot Module Replacement)로 슬라이드 편집 시 즉시 반영.

ES Modules

프로젝트는 "type": "module"을 사용하는 네이티브 ES 모듈 구조입니다.

Markdown

모든 학습 콘텐츠와 슬라이드는 Markdown으로 작성되어 편집이 쉽고 버전 관리에 적합합니다.

package.json 설정

{
  "name": "k6-learn",
  "private": true,
  "version": "0.0.0",
  "type": "module",
  "scripts": {
    "slides": "vite"          // npm run slides 로 개발 서버 시작
  },
  "devDependencies": {
    "reveal.js": "^4.5.0",
    "vite": "^5.0.0"
  }
}

🤝기여 방법

1
이슈 생성

기존에 없는 이슈라면 github.com/grafana/k6-learn/issues에서 새 이슈를 만드세요.

2
저장소 Fork

저장소를 Fork하면 새 브랜치, 변경 사항, 푸시가 원본 저장소에 영향을 주지 않습니다.

3
변경 사항 작성

Fork한 저장소에서 원하는 내용을 수정합니다.

4
Pull Request 생성

Fork에서 원본 저장소로 PR을 생성하고, 해결하는 이슈를 언급합니다. Fork가 최신 상태인지 확인하세요.

5
리뷰 및 병합

PR이 승인되거나 수정 요청을 받게 됩니다. 리뷰어의 피드백을 반영하세요.
